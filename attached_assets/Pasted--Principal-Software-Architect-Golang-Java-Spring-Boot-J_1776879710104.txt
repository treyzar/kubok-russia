Ты — эксперт уровня Principal Software Architect с глубоким опытом в Golang и Java (Spring Boot / Jakarta EE), а также в проектировании высоконагруженных распределённых систем.

Твоя задача — провести полный архитектурный и кодовый анализ backend-проекта на Go (Golang), после чего подготовить план и реализацию его корректного переноса на Java.

## Входные данные
Я предоставлю тебе директорию проекта на Golang. Считай, что это production-сервис.

## Твои цели

### 1. Глубокий анализ проекта (ОБЯЗАТЕЛЬНО)
Перед любыми попытками переписывания ты должен:

- Определить архитектурный стиль:
  (monolith / microservices / clean architecture / hexagonal / layered / etc.)
- Выявить:
  - точки входа (main.go, handlers, routers)
  - структуру модулей и пакетов
  - бизнес-логику
  - инфраструктурные зависимости (DB, cache, message brokers)
- Определить используемые технологии:
  - HTTP-фреймворки (Gin, Echo и т.д.)
  - ORM/DB (GORM, sqlx, raw SQL)
  - middleware
  - concurrency (goroutines, channels)
- Найти:
  - скрытую бизнес-логику
  - неочевидные зависимости
  - паттерны (repository, service, factory и т.д.)

👉 Выведи это в виде структурированного отчёта.

---

### 2. Маппинг Go → Java (КЛЮЧЕВАЯ ЧАСТЬ)

Сформируй таблицу соответствий:

- Go constructs → Java equivalents:
  - goroutines → threads / reactive / CompletableFuture
  - channels → queues / reactive streams
  - interfaces → interfaces (но с учетом DI)
  - error handling → exceptions / Result pattern
  - context.Context → request-scoped context / ThreadLocal / Reactor Context

- Framework mapping:
  - HTTP → Spring Boot (или предложи альтернативу)
  - middleware → filters / interceptors
  - routing → Spring MVC / WebFlux

- Data layer:
  - SQL / GORM → JPA / Hibernate / JDBC / MyBatis (обоснуй выбор)

---

### 3. Архитектурное переосмысление (НЕ ПРОСТО ПЕРЕВОД!)

Ты не должен делать 1:1 перенос.

Ты должен:

- Исправить архитектурные проблемы (если найдёшь)
- Улучшить:
  - масштабируемость
  - читаемость
  - разделение ответственности
- Предложить:
  - новую структуру проекта на Java
  - слои (controller / service / repository / domain)

👉 Объясни каждое архитектурное решение.
