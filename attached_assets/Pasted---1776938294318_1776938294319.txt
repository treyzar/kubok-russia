ТЗ для реплит

Будет очень много правок на фронт. В целом, сценарий использования фронта должен быть следующим:

1. Главную страницу (если пользователь не вошел в систему) оставь формой для авторизации. То есть мы полностью убираем главную страницу для неавторизованных пользователей.
2. Если пользователь авторизовался, то он попадает на главную страницу. На этой главной странице справа должен быть список из возможных игр, в которые может поиграть пользователь. Это текст + иконка. Можешь сделать следующие игры: «Ночной жор» с иконкой холодильника, «Дикий запад» с иконкой поезда, «Тир» с иконкой тира. При выборе определенной игры, справа появляется окно, где можно настроить эту игру. По сути - механика как у сайта stoloto.ru. Для Дикого запада и Тира оставь просто текст «Игра в разработке», а для Ночного жора мы будем реализовывать функционал. Во-первых - это возможность просмотра комнат, в которых ведется игра, и в которые может попасть пользователь. По сути - это ячейки. Их надо красиво оформить. Далее, должен быть механизм фильтрации комнат. Изучи подробно backend часть и пойми, какие фильтры вообще могут быть доступны. Их надо реализовать и на front части. Далее - быстрая игра. Это подбор комнат на основе предыдущих игр пользователя. Если не ошибаюсь, на back части должен быть реализован такой метод. Посмотри. В свою очередь можешь придумать что-то от себя.
3. Никакого создания игры на странице игры (в главной странице) быть не может. Если пользователь администратор, то при нажатии на свой профиль, у него должна возникать кнопка «Админ-панель» в выпадающем списке. При ее нажатии его перебрасывает на админ-панель. Там функционал следующий:  «Админ-панель
Цель админ-панели - предоставить администратору систему максимальные возможности для управления и контролирования работы всей системы.

Панель должна обладать следующими функциональными возможностями:
1. Создание шаблона:
    1. Выбор типа игры (выпадающий список)
    2. Редактирование максимального количества игроков (default = 1)
    3. Редактирование стартового взноса (default = 0)
    4. Редактирование процента куша (default = 100%). 
    5. При изменении любого параметра из пп. 2,3,4 редактируется рядом отображаемая сумма джекпота (она динамически должна обновляться при редактировании параметров)
    6. Редактирование минимального количества игроков (default = 1)
2. При создании шаблона администратору должны быть даны подсказки, насколько рентабельно создавать такой шаблон:
    1. При указании максимального количества игроков = 1, администратору дается подсказка, что это слишком маленькое число игроков
    2. При указании минимального количества игроков > среднее количество реальных игроков во всех комнатах за прошедшую неделю => подсказка, что администратор выставил чересчур большое требование к минимальному количеству игроков
    3. При указании максимального количества игроков > 2 * среднее количество реальных игроков во всех комнатах за прошедшую неделю => подсказка, что администратор выставил чересчур большое требование к максимальному количеству игроков
    4. Стартовый взнос > 1.75 * средняя сумма стартового взноса человека во всех комнатах за прошедшую неделю => подсказка - слишком большое число
    5. Стартовый взнос < 0.5 * средняя сумма стартового взноса человека во всех комнатах за прошедшую неделю => подсказка - слишком маленькое число
    6. Процент куша > 80% => подсказка - слишком большая сумма выдается на джекпот
    7. Процент куша < 50% => подсказка - слишком маленькое число выдается на джекпот
    8. if (есть шаблон с такими же параметрами) => запрет создания шаблона, вывод подсказки
3. Удаление шаблонов. После удаления шаблона, удаляются все комнаты, которые есть в системе. В случае, если в комнате кто-то играет, или набираются игроки => ждать, пока игра закончится, или пока выйдут из набора все игроки и удалить комнату
4. Редактирование шаблонов. Сценарий такой же, как и в удалении, только идет редактирование данных.
5. Статистика:
    1. Главная страница со статистикой содержит список всех шаблонов. Шаблон надо представить как список данных, вводимых при его создании, в одну строку.
    2. Рядом с шаблоном указывается количество сыгранных комнат.
    3. Доступен временной фильтр. Можно выбрать вариант: за час, за день, за неделю, за месяц, за все время, свое время.
    4. Доступна сортировка шаблонов по параметрам.
    5. При нажатии на шаблон, открывается страница с подробной статистикой этого шаблона.
    6. В подробной статистике шаблона указывается:
        1. Количество сыгранных комнат.
        2. Количество реальных игроков.
        3. Количество ботов.
        4. Среднее количество реальных игроков.
        5. Количество игр, которые выиграли реальные игроки.
        6. Количество игр, которые выиграли боты.
        7. Сумма бустов, которая попала в систему из этого шаблона.
        8. Среднее количество бонусов, затрачиваемых на буст (с одного игрока)
        9. Среднее количество бонусов, которые тратят все игроки в одной комнате на бусты.
        10. Среднее количество мест, которые выбирает игрок при игре в комнате.
    7. В подробной статистике шаблона также доступны фильтры и сортировки, которые есть на главной странице статистики.
