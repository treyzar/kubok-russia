# User Flow: Платформа быстрых бонусных игровых комнат

## Обзор системы

Платформа для VIP-аудитории, где пользователи тратят бонусные баллы на участие в коротких real-time раундах и соревнуются за призовой фонд.

---

## Основные роли

- **Пользователь** — участник игровых комнат
- **Администратор** — настраивает параметры комнат через конфигуратор
- **Эксперт** — проверяет журнал раундов и прозрачность результатов

---

## User Flow 1: Список комнат и подбор игры

```
[Пользователь открывает раздел быстрой бонусной игры]
    ↓
[Видит список доступных комнат]
    ↓
[Применяет фильтры и сортировку]
    • Цена входа
    • Количество мест
    • Дополнительные параметры
    ↓
[Система предлагает подходящие комнаты]
    ↓
[Пользователь выбирает комнату]
```

**Ключевые элементы интерфейса:**
- Список комнат с параметрами (цена, места, фонд)
- Фильтры по цене входа и количеству мест
- Сортировка комнат
- Индикация доступности комнаты

---

## User Flow 2: Вход в комнату, резервирование баллов и буст

```
[Пользователь подтверждает участие]
    ↓
[Система резервирует бонусные баллы за вход]
    ↓
[Пользователь попадает в лобби комнаты]
    ↓
    ┌─────────────────────────────────┐
    │ Интерфейс лобби показывает:     │
    │ • Занятые места                 │
    │ • Список участников             │
    │ • Ботов (если есть)             │
    │ • Текущий призовой фонд         │
    │ • Таймер до старта              │
    └─────────────────────────────────┘
    ↓
[До старта: опция купить буст?]
    ├─ ДА → [Покупка буста за бонусные баллы]
    │           ↓
    │       [Отображение вероятности победы с бустом]
    │           ↓
    └─ НЕТ → [Ожидание старта раунда]
```

**Ключевые элементы интерфейса:**
- Подтверждение входа с указанием стоимости
- Индикатор резервирования баллов
- Лобби с real-time обновлениями
- Кнопка покупки буста
- Визуализация вероятности победы

## User Flow 3: Таймер, заполнение комнаты и real-time запуск

```
[Первый пользователь входит в комнату]
    ↓
[Запускается таймер ожидания: 1 минута]
    ↓
    ┌──────────────────────────────────────┐
    │ Real-time обновления в лобби:        │
    │ • Новые участники присоединяются     │
    │ • Обновление занятых мест            │
    │ • Изменение призового фонда          │
    │ • Отсчёт таймера                     │
    └──────────────────────────────────────┘
    ↓
[Таймер истёк]
    ↓
[Свободные места заполняются ботами]
    ↓
    ┌──────────────────────────────────────┐
    │ Финальный состав отображается:       │
    │ • Реальные участники (с иконкой)     │
    │ • Боты (явно помечены)               │
    │ • Итоговый призовой фонд             │
    └──────────────────────────────────────┘
    ↓
[Статус комнаты: ЗАПУСК РАУНДА]
```

**Ключевые элементы интерфейса:**
- Таймер обратного отсчёта
- Real-time список участников
- Визуальное различие между игроками и ботами
- Индикатор заполненности комнаты
- Анимация перехода к раунду

## User Flow 4: Определение победителя и визуальный раунд

```
[Раунд запущен]
    ↓
    ┌──────────────────────────────────────┐
    │ Backend обрабатывает:                │
    │ • Параметры комнаты                  │
    │ • Список участников и их места       │
    │ • Примененные бусты                  │
    │ • RNG / взвешенный алгоритм          │
    └──────────────────────────────────────┘
    ↓
[Backend определяет победителя]
    ↓
[Frontend показывает визуальную оболочку]
    • Гонка / Арена / Карты / Колесо / и т.д.
    ↓
[Визуализация процесса игры]
    • Видны все участники
    • Видны боты
    • Отображается призовой фонд
    ↓
[Финал визуальной оболочки]
    ↓
[Объявление победителя]
    ↓
    ┌──────────────────────────────────────┐
    │ Экран результатов:                   │
    │ • Победитель                         │
    │ • Выигрыш                            │
    │ • Изменение баланса пользователя     │
    │ • Быстрые действия:                  │
    │   - Повторить игру                   │
    │   - Похожая комната                  │
    │   - Более рискованная комната        │
    └──────────────────────────────────────┘
```

**Ключевые элементы интерфейса:**
- Визуальная оболочка раунда (выбор команды)
- Индикация всех участников и ботов
- Отображение призового фонда
- Анимация определения победителя
- Экран результатов с быстрыми действиями

## User Flow 5: Конфигуратор (Администратор)

```
[Администратор открывает конфигуратор]
    ↓
    ┌──────────────────────────────────────┐
    │ Настраиваемые параметры:             │
    │ • Количество мест в комнате          │
    │ • Цена входа (бонусные баллы)        │
    │ • Параметры призового фонда          │
    │ • Дополнительные настройки           │
    └──────────────────────────────────────┘
    ↓
[Администратор изменяет параметры]
    ↓
    ┌──────────────────────────────────────┐
    │ Система показывает в real-time:      │
    │                                      │
    │ ✓ Привлекательность для пользователя │
    │   • Ожидаемая ценность игры          │
    │   • Вероятность выигрыша             │
    │                                      │
    │ ✓ Выгодность для организатора        │
    │   • Маржинальность                   │
    │   • Оборот бонусных баллов           │
    │                                      │
    │ ⚠ Предупреждения о проблемах         │
    │   • Слишком высокий % фонда          │
    │   • Нереалистичная цена              │
    │   • Дорогой буст                     │
    └──────────────────────────────────────┘
    ↓
[Валидация конфигурации]
    ├─ Валидна → [Сохранение настроек]
    │                ↓
    │            [Комната создана/обновлена]
    │
    └─ Невалидна → [Показ ошибок и предупреждений]
                       ↓
                   [Возврат к редактированию]
```

**Ключевые элементы интерфейса:**
- Форма настройки параметров комнаты
- Real-time расчёт метрик
- Визуализация привлекательности и выгодности
- Система предупреждений
- Кнопки сохранения/отмены

## User Flow 7: Невалидная конфигурация комнаты (Администратор)

```
[Администратор в конфигураторе]
    ↓
[Ввод параметров комнаты]
    ↓
[Real-time валидация]
    ↓
    ┌──────────────────────────────────────┐
    │ Проверка на проблемные настройки:    │
    │                                      │
    │ ⚠ Слишком высокий % фонда победителю │
    │   (> 90% → низкая маржинальность)    │
    │                                      │
    │ ⚠ Слишком дорогой буст               │
    │   (> 50% от входа → низкий спрос)    │
    │                                      │
    │ ⚠ Нереалистичная цена входа          │
    │   (> средний баланс пользователей)   │
    │                                      │
    │ ⚠ Противоречивые параметры           │
    │   (фонд < сумма входов)              │
    └──────────────────────────────────────┘
    ↓
    Есть критические ошибки?
    ├─ ДА → [Блокировка сохранения]
    │           ↓
    │       ┌──────────────────────────────────┐
    │       │ "Невозможно сохранить"           │
    │       │ • Список критических ошибок      │
    │       │ • Рекомендации по исправлению    │
    │       └──────────────────────────────────┘
    │           ↓
    │       [Возврат к редактированию]
    │
    └─ НЕТ → Есть предупреждения?
                ├─ ДА → [Показ предупреждений]
                │           ↓
                │       ┌──────────────────────────────────┐
                │       │ "Сохранить с предупреждениями?"  │
                │       │ • Список рисков                  │
                │       │ • Возможные последствия          │
                │       └──────────────────────────────────┘
                │           ↓
                │       [Подтверждение администратора]
                │           ├─ ДА → [Сохранение]
                │           └─ НЕТ → [Возврат к редактированию]
                │
                └─ НЕТ → [Успешное сохранение]
```

**Ключевые элементы интерфейса:**
- Real-time индикаторы валидации
- Цветовая кодировка (ошибки/предупреждения)
- Подробные объяснения проблем
- Рекомендации по исправлению
- Блокировка кнопки "Сохранить" при критических ошибках

## Ключевые принципы UX

1. **Real-time обновления** — все изменения отображаются мгновенно
2. **Прозрачность** — пользователь всегда видит свой баланс, шансы, состав комнаты
3. **Быстрая реиграбельность** — минимум кликов до следующей игры
4. **Понятные ошибки** — чёткие сообщения с предложением решения
5. **Визуальная обратная связь** — анимации, индикаторы, подтверждения действий
6. **Различие игроков и ботов** — явная маркировка
7. **Доступность** — интуитивная навигация для всех групп пользователей

---

## Технические требования к интерфейсу

- **WebSocket** для real-time обновлений
- **Адаптивный дизайн** (опционально на MVP)
- **Индикаторы загрузки** для всех асинхронных операций
- **Обработка ошибок сети** с возможностью повтора
- **Кэширование** списка комнат для быстрого отображения
- **Оптимистичные обновления UI** для улучшения отзывчивости
